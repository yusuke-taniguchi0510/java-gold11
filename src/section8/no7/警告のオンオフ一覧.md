# 警告のON・OFF

| 非推奨のメソッドの宣言側                     | 非推奨のメソッドの呼び出し側                | 警告のON/OFF |
|----------------------------------|-------------------------------|-----------|
| コンパイル時に警告の出るコード                  | @SuperWarningsなし              | ON        |
| コンパイル時に警告の出るコード                  | @SuperWarnings("unchecked")   | OFF       |
| @Deprecated                      | @SuperWarningsなし              | ON        |
| @Deprecated                      | @SuperWarnings("deprecation") | OFF       |
| @Deprecated(forRemoval = "true") | @SuperWarnings("deprecation") | ON        |
| @Deprecated(forRemoval = "true") | @SuperWarnings("removal")     | OFF       |

<br>
<br>
※以下のように警告条件が複数の場合は、抑制側も複数指定可能

```java
public class Sample {
    @Deprecated
    public void sample() {
        System.out.println("sample");
    }
}

public class Main {

    @SuppressWarnings({"unchecked", "deprecation"}) // このメソッド自体がコンパイル時に警告対象かつDeprecatedのコードを使用しているため
    public static void main(String[] args) {
        List list = new ArrayList(); // ジェネリクスなしで初期化
        list.add("hello"); // Stringをadd　※ここでコンパイル時警告の対象となる
        new Sample().sample();
    }
}
```
